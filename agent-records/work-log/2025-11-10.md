#### 3. Architecture Refactoring: Layered Design

Refactored Slice 0 (`explain` command) to use clean architecture with proper separation of concerns:

**Domain Layer** (`src/graft/domain/`):
- Created `entities.py` with core domain entities:
  - `Artifact` - Main domain entity representing a graft artifact
  - `GraftConfig` - Complete graft.yaml configuration
  - `Derivation` - Single derivation specification
  - `Inputs`, `Material`, `Output`, `Template`, `Policy` - Value objects

**Adapter Layer** (`src/graft/adapters/`):
- Created `filesystem.py`:
  - `FileSystemPort` protocol defining file I/O interface
  - `LocalFileSystem` implementation for local file operations
- Created `config.py`:
  - `ConfigAdapter` for loading and parsing `graft.yaml` files
  - Converts raw YAML to domain entities

**Service Layer** (`src/graft/services/`):
- Created `explain.py`:
  - `ExplainService` orchestrating artifact explanation use case
  - `ExplainResult` result object with `.to_dict()` for serialization
  - Clean separation from presentation concerns

**CLI Layer** (`src/graft/cli.py`):
- Refactored `explain` command to delegate to `ExplainService`
- Dependency injection: wires up adapters and services at module level
- Thin presentation layer handling output formatting and error messages

### Key Decisions

1. **Immutable domain entities**: Used `@dataclass(frozen=True)` for value objects to prevent accidental mutation
2. **Protocol-based ports**: Used `typing.Protocol` for adapter interfaces to enable static type checking without runtime overhead
3. **Explicit dependency injection**: Services receive dependencies via constructor rather than global state
4. **Result objects**: Services return domain-specific result objects with serialization methods
