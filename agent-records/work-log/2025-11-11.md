#### 2. Service Layer Implementation

**Created ImpactService** (`src/graft/services/impact.py`):
- Follows layered architecture pattern from ADR-0002
- `ImpactService.impact(artifact_path)` returns `ImpactResult`
- Result contains `artifact` path and `downstream` array
- Current implementation returns empty downstream list (stub for future dependency graph analysis)
- Clean separation of business logic from CLI concerns

**Created SimulateService** (`src/graft/services/simulate.py`):
- Implements dry-run simulation without side effects
- `SimulateService.simulate(artifact_path, cascade=bool)` returns `SimulateResult`
- Returns confirmation message indicating simulation completion
- Stub implementation - future versions will validate inputs, check templates, and simulate downstream builds

**Design Philosophy**:
- Both services follow the same pattern as `ExplainService`, `RunService`, `StatusService`
- Result objects with `.to_dict()` methods for JSON serialization
- No dependencies on adapters (future versions may add dependency graph analysis)
- Stub implementations satisfy Slice 4 acceptance criteria

#### 3. CLI Integration

**Updated `src/graft/cli.py`**:
- Added imports for `ImpactService` and `SimulateService`
- Initialized services via dependency injection
- Refactored `impact` command to use `ImpactService`:
  - Proper error handling with exit codes (0=success, 1=user error, 2=system error)
  - JSON output via `--json` flag
  - Human-readable fallback (JSON formatting for now)
- Refactored `simulate` command to use `SimulateService`:
  - Prints confirmation message from service
  - Respects `--cascade` flag
  - Proper error handling

#### 4. JSON Schema

**Created `schemas/cli/impact.schema.json`**:
- Follows JSON Schema Draft 2020-12
- Defines contract for `graft impact --json` output
- Properties: `artifact` (string), `downstream` (array of objects)
- Downstream objects can include `artifact` path and `relationship` type
- Matches pattern of `status.schema.json`
